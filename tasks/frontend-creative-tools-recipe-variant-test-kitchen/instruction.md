Frontier-loss hypothesis

Deterministic fixture

The fictional base recipe is a 900-gram Cedar Seed Loaf with 12 ingredients, seven preparation steps, four equipment resources, six substitution candidates, fixed nutrition/allergen/cost values, two existing test batches, and deterministic evaluator thresholds. Units are grams, milliliters, Celsius, and minutes; no network or food advice is involved.

Hardening update (written per-proposal from trial evidence, 2026-07-23)

Additive to the v1 contract above, specific to this proposal's baker's-percentage formulation, substitution propagation, and immutable batch-evidence design. Evidence: gpt-5.6-sol xhigh fleet plus the healthy provenance/claude-sonnet-5 builder sweep (8 trials, mean 0.8833 — Sonnet 5 fails these contracts quietly at 0.84+ scores).

1. Substitution propagates through all seven consequence channels in one transition. Substituting one ingredient must update formulation mass, hydration (baker's percentage recomputed against flour weight, exact), allergen flags, cost, process steps, predictions, and the artifact together, with the previous values gone from every surface — formulation graph, percentage table, step timeline, batch ledger, sensory radar, and variant tree. Trial evidence: stale linked views after mutation are the most frequent Sonnet-5 miss at 0.84+ scores; a substitution that updates the percentage table but leaves the step timeline or predictions stale is exactly the shallow build.
2. Mass conservation is the spine — criterion-ize it. Every formulation edit conserves total mass to the gram with the rounding rule for rational baker's percentages stated explicitly (integer grams, rounding direction named); the percentage table, batch ledger, and scaled variants all reconcile to the same conserved total; a scaling operation preserves ratios exactly rather than rounding independently per ingredient (state the exact rounding contract). Trial evidence: conservation-style invariants (component weights + residue = parent in the mass-flow task family) are where plausible builds drift silently.
3. Prior batch evidence is immutable. "Preserving prior batch evidence" needs the full contract: recorded test batches, observations, and sensory scores are never rewritten by later substitutions or formula edits; the variant tree links each batch to the exact formula version it was baked from; deleting a variant branch removes only that branch's unexecuted state, never executed-batch records; the evaluator cites the exact formula version per finding.
4. Allergen transitions never clear silently. An allergen flag introduced by substitution is visible on every surface that shows the formula or its batches (formulation graph, ledger, handoff artifact) and can only be removed by a substitution that genuinely eliminates it, with the change recorded in history — a safety-relevant state that a shallow build folds into a summary is a first-class defect.
5. Test-batch execution recovery is exactly-once. Failed or interrupted batch runs preserve completed observations; retrying reuses the same inputs and attempt history, increments attempt counts exactly once per retry (rapid double-retry = one added attempt), and never duplicates a recorded observation; the partial-run state is named and visible.
6. Deterministic evaluator and approval effects. Evaluator findings are deterministic for the same formula + batch set and cite exact versions; approving a formula for handoff is graded on post-confirm state (status flips, handoff artifact reflects the approved version, unapproved variants stay excluded) — never the dialog's presence (trial evidence: the 0.9636 Sonnet-5 run failed four criteria because confirming did nothing). Compare mode shows both variants' radar axes and deltas from one shared state.
7. Motion numerics and a testable reduced-motion path. Substitution-propagation, radar-morph, and timeline-shift transitions need named durations (150–300ms) with early/settled frame sampling and computed hover deltas on nodes and table cells; reduced motion is verifiable via a visible chrome toggle or ?reducedMotion=1 fresh load — never browser prefers-reduced-motion emulation (emulation-dependent criteria BLOCKED in trial).
8. Cross-model pilot calibration. After the oracle is certified: three independent healthy gpt-5.6-sol xhigh trials and three provenance/claude-sonnet-5 trials on the identical revision, reporting both expected failure signatures — GPT: hydration math drift after chained substitutions, scaling that rounds per-ingredient and breaks conservation, artifact schema drift; Sonnet 5: allergen flags not propagating to the handoff artifact, predictions stale after substitution, approval that never gates handoff while reporting completion. Keep/deepen/clarify/rework when misses are mostly infrastructure, blocked judging, or ambiguity.
