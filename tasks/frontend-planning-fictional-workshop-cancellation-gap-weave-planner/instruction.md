# Fictional Workshop Cancellation Gap-Weave Planner

<summary>
Build a framework-agnostic frontend application for filling a cancellation gap in one fictional workshop day. The coordinator drags waitlist request WL-07 into exact vacancy GAP-04, sees the candidate fit the bench but collide with facilitator Ivo flexible booking BK-12, previews and cancels a deterministic one-booking ripple, then repeats and confirms it. The calendar, waitlist lattice, facilitator strip, tool graph, offer clock, utilization lens, event history, WebMCP state, and exact calendar packet must update from one canonical schedule model. This is not a booking form, customer CRM, generic calendar, salon website, Kanban board, or resource table. It does not accept real appointments, deposits, messages, personal details, or external calendar writes. Its defining browser interaction is cancellation-gap weaving. A waitlist card must occupy an exact visual vacancy while one eligible flexible booking moves to its only valid later position, and every linked representation must preserve the same minutes, resource claims, offer state, selection, provenance, and export semantics.
</summary>

Feature bounded ripple preview cancel and atomic weave

Preview reports the complete move set before confirmation. One infeasible descendant, shared-resource collision, operating-window breach, or third required move blocks commit and suggests a different candidate/gap. Cancel is a total no-op. Confirm consumes a single-use preview token and appends one atomic weave event containing the request plus exact move set. If any expected revision or resource hash changed, confirmation rejects and offers a fresh preview without mutation. Double activation, repeated Enter, pointer-up after keyboard confirm, or back-button re-entry cannot append another event.
