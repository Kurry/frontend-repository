<summary>
Build a frontend-only Ghostty terminal configuration generator with a live preview, using Svelte 5 with runes and stores, Tailwind CSS 4.3.2, and Bits UI. The app opens directly into a categorized settings editor whose controls drive a live terminal preview and a monospace generated-config output in exact sync. The app produces the user's Ghostty config file — a downloadable and copyable key = value artifact compiled live from the session overrides.
</summary>

<core_features>
Core features (each line is an observable behavior the finished app must exhibit):

Feature: Settings editor shell —
- The first usable view is the configuration editor, not a welcome or marketing page: a left sidebar of setting categories (Application, Terminal, Clipboard, Window, Colors, Fonts, Keybinds, Mouse, GTK, Linux, macOS) beside a grouped settings form, with a generated-configuration preview visible in the same view.
- Navigating between setting categories keeps the current category visibly identified and does not reload the document.
- A settings search field filters visible setting rows by label or key as the user types; clearing the query restores the full unfiltered form for the active category.
- Opening the command palette with Cmd+K or Ctrl+K, typing a fuzzy match, and choosing a result navigates to that setting's category, focuses the matching control, and closes the palette without a document reload.

Feature: Live generated Ghostty config artifact —
- The generated-config panel renders the current overrides as monospace key = value lines under the header lines "# Config generated by Ghostty Config" and "# https://ghostty.zerebos.com", shows an override count, and shows "# No changes to the default config yet" when nothing has been changed.
- Changing the main font-family control updates the generated configuration immediately: the generated line reads font-family = the entered value, and the control value, the live preview, and the generated line always agree.
- Changing the base font size (a slider defaulting to 13, range 4 to 60) updates the generated font-size line and the live preview text size.
- Changing a cursor setting updates the live cursor preview: the cursor-style control (block, bar, underline, hollow block) swaps the previewed cursor shape, and the generated cursor-style line follows using block, bar, underline, or block_hollow (hollow block maps to block_hollow).
- Editing base colors (background default #282c34, foreground default #ffffff), the 16-color palette, window padding, and background opacity each update the generated configuration and the live preview.
- Selecting a color theme on the Colors page recolors the live terminal preview and adds a theme = the chosen theme line to the generated configuration. Themes offered are exactly: Dracula; Nord; Gruvbox Dark; Catppuccin Mocha; Solarized Dark - Patched; Builtin Dark.
- Copy writes the generated configuration to the clipboard and shows visible feedback without altering the configuration text; Download saves a file named ghostty-config; Reset all clears every override back to defaults. These actions are disabled while there are zero overrides. Copy and Download MUST contain every session override as key = value lines — an export that omits session mutations is invalid.
- An Import control accepts pasted Ghostty config text (key = value lines plus # comments). Applying a valid paste sets matching overrides so the generated panel, controls, live preview, and override count all reflect the imported values. Malformed or contract-invalid Import shows a visible parse or validation error, applies no overrides, and leaves the override count unchanged.

Feature: Ghostty create/update field contracts —
- The generated config is the would-be Ghostty create/update request body: each committed override is a key = value line matching these contracts. Valid create/update and Import produce that same body; Export emits conforming lines containing the session's actual overrides.
- Required shapes when a key is overridden: font-family non-empty string; font-size integer 4 to 60 inclusive; cursor-style exactly one of block, bar, underline, block_hollow; background, foreground, and cursor-color as #RRGGBB hex; background-opacity and cursor-opacity numbers from 0 to 1 inclusive; theme exactly one of Dracula, Nord, Gruvbox Dark, Catppuccin Mocha, Solarized Dark - Patched, Builtin Dark; clipboard-read and clipboard-write each exactly one of ask, allow, deny; copy-on-select exactly one of true, false, clipboard; window-decoration exactly one of auto, none, client, server; window-padding-x and window-padding-y non-negative numbers; scrollback-limit non-negative integer; palette lines as N=#RRGGBB with N an integer 0 to 255; keybind lines as trigger=action; boolean settings as true or false; profile-name a non-empty string when saving a profile.
- Cross-field rules: duplicate keybind triggers show a conflict notice naming both actions and the notice clears when the duplicate is resolved; an out-of-bound or out-of-enum value shows an inline message naming that setting next to the control and no malformed line is emitted into the generated configuration while the value is invalid; Import of contract-invalid text is rejected the same way without mutating overrides.

Feature: Undo, Diff, and named profiles —
- Undo and Redo reverse and reapply setting mutations; after editing font-family then undoing, the generated line, override count, and live preview revert; Redo restores the edit on all three surfaces. Each control disables when its stack is empty.
- A Diff mode toggle shows each overridden key's default and current values side by side with a distinct changed treatment; with zero overrides it shows an explicit no-differences state; toggling Diff does not clear overrides.
- Save profile with a required non-empty name stores the current override set; selecting that profile after Reset all re-applies the same overrides to the editor, preview, and generated config; Delete removes the profile after confirmation.
- Each saved profile row shows its name and its stored override count, and exposes a Rename control that opens an inline form pre-filled with the current name; the same non-empty trimmed name contract applies, a name duplicating another saved profile is rejected with a message naming the conflict, and a valid rename updates the profile's name in the list without changing its stored overrides.
- After applying a valid Import, Diff mode lists every imported key side by side with its default value and the imported current value in the changed treatment, so an imported existing config is reviewable against defaults.

Feature: Theme A/B compare —
- A Compare themes control on the Colors page opens a side-by-side surface with two live terminal previews labelled A and B, each with its own theme selector offering exactly the six themes; changing one side's selection recolors only that side's preview immediately.
- Both previews render the same sample terminal content so visible differences are attributable to the themes alone, and the compare surface never changes the generated configuration or the override count until an Apply is chosen.
- Each side offers an Apply control: applying a side commits that side's theme as the theme override — the generated theme = line, the main live preview, and the override count update exactly as if the theme had been picked directly; closing the compare surface without applying leaves overrides unchanged.
</core_features>

<user_flows>
- Editing the font family end to end: entering a new font-family value updates the generated font-family line to the entered value, increases the override count by exactly one, and re-renders the live terminal preview; switching to another category and back shows the same value in the control, the preview, and the generated line without a reload.
- Selecting a theme end to end: choosing a theme on the Colors page recolors the live terminal preview, adds a theme = the chosen theme line to the generated configuration, and updates the override count; the export actions become enabled if they were not already.
- Resetting a single override: resetting the edited font-family setting removes its generated line (or shows it empty), decreases the override count by exactly one, and returns the preview text to the default face, while every unrelated override stays unchanged in the panel and the preview.
- Reset all end to end: after editing at least two settings in different categories, Reset all removes every generated override line, returns the override count to zero, restores the "# No changes to the default config yet" state, restores the live preview to defaults, and disables the export actions.
- Reloading the page restores the same view and the previously edited settings from client storage: the edited controls, the live preview, the generated lines, the override count, and saved profiles all match their pre-reload state.
- Export then Import round-trip: after editing at least two settings, Copy or Download the generated config, Reset all to empty, then Import that same text — controls, live preview, generated lines, and override count all match the pre-reset state. An export that omitted those mutations would fail this flow.
- Undo/Redo round-trip: edit font-family, confirm the override appears, Undo once (override and count revert), then Redo once — the override returns on the control, preview, and generated line.
- Command palette jump: open the palette with Cmd+K or Ctrl+K, type enough of a setting name to match, choose the result, and land on that setting's category with the control focused.
- Profile save and apply: with at least two overrides, Save profile under a name, Reset all, then select that profile — the same overrides reappear in the generated panel, controls, and preview.
- Profile rename flow: save a profile, rename it via its Rename control to a new valid name — the list shows the new name with the same stored override count, and applying the renamed profile still re-applies the same overrides; attempting to rename it to another saved profile's name shows the conflict message and leaves both names unchanged.
- Theme A/B flow: open Compare themes, select two different themes for sides A and B — the two previews recolor differently; choosing Apply on side B adds theme = side B's theme to the generated configuration, recolors the main live preview, and updates the override count; reopening compare and closing it without applying leaves the generated configuration and count identical.
- Import-then-diff flow: from zero overrides, Import a config text containing at least two in-contract keys, then open Diff mode — each imported key appears with its default value beside the imported value in the changed treatment, and the override count equals the number of imported keys.
- Field-contract validation flow: attempting to commit an out-of-contract value (for example font-size outside 4 to 60, cursor-style outside the closed enum, a non-#RRGGBB color, or an empty profile-name) shows inline validation naming the field, leaves the generated configuration without that malformed line, does not change the override count for the rejected value, and keeps the rest of the workflow operable without a reload; a subsequent valid in-contract commit emits the matching key = value line.
</user_flows>

<edge_cases>
- Entering two different font-family values in succession leaves the generated line following the latest value, with no duplicate font-family lines in the output.
- Deleting or resetting one override removes exactly that override; every unrelated override keeps its line and value.
- With zero overrides, the generated-config panel shows the empty-state comment line, the override count reads zero, and Copy, Download, and Reset all are disabled and read as non-interactive.
- The font-size control cannot produce a value outside 4 to 60: dragging stops at the bounds, and a typed out-of-range value is rejected or clamped with the shown value staying inside the range.
- Copying does not mutate state: after Copy, the generated configuration text and the override count are identical to their pre-copy values.
- Importing malformed config text (lines that are not comments and not valid key = value) shows a visible parse error naming the problem, applies no overrides, and leaves the override count unchanged.
- Importing or committing a value that violates a Ghostty field contract shows inline validation naming the field, does not emit that malformed override, and on Import leaves existing overrides unchanged.
- Undo with an empty history and Redo with an empty redo stack are disabled, not silent no-ops.
- Saving a profile with an empty or whitespace-only name creates no profile and shows validation naming the name field.
- On the Keybinds page, assigning the same shortcut trigger to two different actions shows a conflict notice naming both actions; the notice clears when the duplicate trigger is resolved.
- Renaming a profile to another saved profile's name is rejected with a message naming the conflict, leaving both profiles' names and overrides unchanged; renaming a profile to its own current name is not treated as a conflict.
- In Compare themes, selecting the same theme on both sides renders two matching previews, and Apply still commits exactly one theme = line with no duplicates.
- Closing the compare surface without choosing Apply leaves the generated configuration text and the override count byte-identical to their pre-open values.
</edge_cases>

<requirements>
- Use Svelte 5 with runes and stores, and Tailwind CSS 4.3.2 (pinned) as the styling base. Keep the implementation frontend-only and self-contained; do not depend on a live backend.
- No authentication wall — open directly into the primary workspace (the categorized settings editor with its generated-config preview), not a welcome or framework default page.
- Shared application state (the settings values, the active category, derived generated-config output, undo/redo stacks, Diff mode, the compare surface's A and B theme selections, profiles, and search/palette query) lives in Svelte stores/runes as the single source of truth; views derive from that one store. The control value, the live preview, and the generated configuration line for a setting must always agree. WebMCP tool handlers invoke the same store commands as the visible controls.
- Use Bits UI as the component library for interactive primitives: dropdowns/selects, switches, toggles, sliders, dialogs, tooltips, and toast surfaces build on it rather than hand-rolled equivalents.
- AutoAnimate and Svelte transitions are allowed for animation; no other animation libraries.
- Phosphor icons only, via the phosphor-svelte package; one icon set used consistently across the app.
- All forms and validated setting editors — including the profile save and rename name forms and Import — are driven by a form library (TanStack Form for Svelte or Felte) paired with a Zod schema that mirrors the Ghostty create/update field contracts above (keys, types, bounds, enums, cross-field rules). Inline per-field errors appear before any value is committed. A valid commit produces the would-be request body; Export and Import validate through the same schema.
- All libraries are installed via npm and bundled locally; no CDN imports of any library, font, or icon set.
- Persist relevant state in client storage (localStorage or equivalent) so a reload restores the edited settings, the same view, and saved profiles.
- Seed the editor so the primary workflow is usable on first load with sensible defaults (font-size 13, background #282c34, foreground #ffffff, the standard 16-color palette, and the default keybind set), and so the generated-config panel starts from a known empty state.
- End-state contract: provide the generated-configuration output as monospace key = value text with the two header comment lines and an override count, an empty state when nothing is overridden, and Copy, Download, Import, and Reset all actions. Copy and Download MUST reflect the session's actual overrides; Import MUST restore the same visible state (round-trip). An export that omits session work is invalid.
- Deleting or resetting an override must remove exactly that override; it must not clear or alter unrelated overrides.
- Do not fabricate success: Copy/Download/Reset/Import/Undo/Redo/profile apply and every setting edit must reflect real state, differ for different inputs, and survive a reload.
</requirements>
