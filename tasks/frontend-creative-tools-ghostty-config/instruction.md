<core_features>
Core features (each line is an observable behavior the finished app must exhibit):
- The first usable view is the configuration editor, not a welcome or marketing page: a left sidebar of setting categories (Application, Terminal, Clipboard, Window, Colors, Fonts, Keybinds, Mouse, GTK, Linux, macOS) beside a grouped settings form, with a generated-configuration preview visible in the same view.
- The generated-config panel renders the current overrides as monospace key = value lines under the header lines "# Config generated by Ghostty Config" and "# https://ghostty.zerebos.com", shows an override count, and shows "# No changes to the default config yet" when nothing has been changed.
- Changing the main font-family control updates the generated configuration immediately: the generated line reads font-family = the entered value, and the control value, the live preview, and the generated line always agree.
- Changing the base font size (a slider defaulting to 13, range 4 to 60) updates the generated font-size line and the live preview text size.
- Changing a cursor setting updates the live cursor preview: the cursor-style control (block, bar, underline, hollow block) swaps the previewed cursor shape, and the generated cursor-style line follows.
- Editing base colors (background default #282c34, foreground default #ffffff), the 16-color palette, window padding, and background opacity each update the generated configuration and the live preview.
- Selecting a color theme on the Colors page recolors the live terminal preview and adds a theme = the chosen theme line to the generated configuration.
- Copy writes the generated configuration to the clipboard and shows visible feedback without altering the configuration text; Download saves a file named ghostty-config; Reset all clears every override back to defaults. These actions are disabled while there are zero overrides.
- Typing an invalid value into a validated setting control shows an inline message naming that setting next to the control, and no malformed line is emitted into the generated configuration while the value is invalid.
- Navigating between setting categories keeps the current category visibly identified and does not reload the document.
</core_features>

<user_flows>
- Editing the font family end to end: entering a new font-family value updates the generated font-family line to the entered value, increases the override count by exactly one, and re-renders the live terminal preview; switching to another category and back shows the same value in the control, the preview, and the generated line without a reload.
- Selecting a theme end to end: choosing a theme on the Colors page recolors the live terminal preview, adds a theme = the chosen theme line to the generated configuration, and updates the override count; the export actions become enabled if they were not already.
- Resetting a single override: resetting the edited font-family setting removes its generated line (or shows it empty), decreases the override count by exactly one, and returns the preview text to the default face, while every unrelated override stays unchanged in the panel and the preview.
- Reset all end to end: after editing at least two settings in different categories, Reset all removes every generated override line, returns the override count to zero, restores the "# No changes to the default config yet" state, restores the live preview to defaults, and disables the export actions.
- Reloading the page restores the same view and the previously edited settings from client storage: the edited controls, the live preview, the generated lines, and the override count all match their pre-reload state.
</user_flows>

<edge_cases>
- Entering two different font-family values in succession leaves the generated line following the latest value, with no duplicate font-family lines in the output.
- Deleting or resetting one override removes exactly that override; every unrelated override keeps its line and value.
- With zero overrides, the generated-config panel shows the empty-state comment line, the override count reads zero, and Copy, Download, and Reset all are disabled and read as non-interactive.
- The font-size control cannot produce a value outside 4 to 60: dragging stops at the bounds, and a typed out-of-range value is rejected or clamped with the shown value staying inside the range.
- Copying does not mutate state: after Copy, the generated configuration text and the override count are identical to their pre-copy values.
</edge_cases>

<visual_design>
- A desktop, macOS-style application window: a translucent left sidebar with window-dot chrome and category icons, a titled content area with a back/forward history control, and a right-hand generated-config panel — a clear two-pane settings-and-preview composition.
- The settings form uses grouped sections with section titles and notes, labeled rows, and a mix of control types: sliders, number steppers with unit toggles, text inputs, dropdowns, pill toggles, switches, color pickers, a palette grid, and repeatable-value list editors.
- The generated-config output is monospace, uses a terminal color scheme (dark background, light text, colorized keys and values), and reads as terminal configuration rather than generic cards.
- The first viewport gives greater visual emphasis to the categorized form and its generated-config preview than to secondary metadata, notes, or badges.
- The font-family change produces a distinguishable updated state (the new generated line, the override count, the reset affordance) without overlapping or jumping content.
- A macOS-style dock hosts a live interactive terminal preview that re-themes as settings change.
- Sidebar categories and control affordances use one consistent icon style throughout; no mixed icon weights or mismatched glyph sets.
- Light and dark surfaces stay legible; status colors are consistent (danger/reset red, success green).
</visual_design>

<motion>
- Controls give immediate visual feedback on interaction: sliders track the drag, switches and pill toggles animate their selected state, and buttons show a pressed treatment on pointer-down.
- Focused controls show a clearly visible focus ring distinct from hover.
- The current category and the current selection are visually identifiable at every step, including the active sidebar item.
- Category switches animate the content in without a full page reload; the content header stays put while the body transitions.
- A new override line entering the generated-config panel animates in, and a removed override line animates out, without surrounding lines jumping or overlapping.
- The interactive terminal preview re-themes live as colors, fonts, cursor, and palette are edited.
- Copy, download, and reset produce transient toast feedback that enters with motion, stays readable, and dismisses with a fade; the cursor preview blinks for emphasis.
- Hover, focus, pressed, selected, and disabled states are visually distinct and consistent across controls of the same kind; disabled export buttons read as non-interactive while their labels stay legible.
- With prefers-reduced-motion set, transitions are removed or minimized and every state change still applies instantly and completely.
</motion>

<responsiveness>
- At desktop widths the sidebar, settings form, and generated-config panel are visible together as the two-pane composition with the sidebar alongside.
- The layout reflows at a narrow width (about 375 pixels): the panes stack and controls stay operable without horizontal page scrolling while keeping the product identity.
</responsiveness>

<accessibility>
- The font-family input can be reached and edited with the keyboard alone, and typed text updates both the input and the generated font-family line.
- Every interactive control — sidebar categories, sliders, switches, pill toggles, dropdowns, and the export actions — is reachable and operable with the keyboard, with a visible focus indicator on each.
- Switches and pill toggles expose their on/off or selected state to assistive technology, and that state updates when toggled.
- Copy, download, and reset feedback is announced through a polite live region as well as shown visually.
</accessibility>

<performance>
- The app is interactive within 2 seconds of a local cold load.
- No console errors or warnings appear during a full exercise of the editor, exports, and category navigation.
- Dragging a slider or rapidly toggling switches updates the live preview and generated configuration without hangs, dropped interactions, or visible lag.
</performance>

<writing>
- Microcopy is concise sentence-case UI text with numerals for counts; section notes explain what a setting does in one short sentence.
- Validation messages name the setting and the accepted range or format; no placeholder or lorem text appears anywhere in the shipped UI.
- The generated output reads as terminal configuration: comment lines start with #, and every override renders as a key = value line.
</writing>

<requirements>
- Use Svelte 5 with runes and stores, and Tailwind CSS 4.3.2 (pinned) as the styling base. Keep the implementation frontend-only and self-contained; do not depend on a live backend.
- No authentication wall — open directly into the primary workspace (the categorized settings editor with its generated-config preview), not a welcome or framework default page.
- Shared application state (the settings values, the active category, and derived generated-config output) lives in Svelte stores/runes as the single source of truth; views derive from that one store. The control value, the live preview, and the generated configuration line for a setting must always agree.
- Use Bits UI as the component library for interactive primitives: dropdowns/selects, switches, toggles, sliders, dialogs, tooltips, and toast surfaces build on it rather than hand-rolled equivalents.
- AutoAnimate and Svelte transitions are allowed for animation; no other animation libraries.
- Phosphor icons only, via the phosphor-svelte package; one icon set used consistently across the app.
- All forms and validated setting editors are driven by a form library (TanStack Form for Svelte or Felte) paired with a Zod schema: the schema defines the accepted values and ranges, and inline per-field errors appear before any value is committed.
- All libraries are installed via npm and bundled locally; no CDN imports of any library, font, or icon set.
- Persist relevant state in client storage (localStorage or equivalent) so a reload restores the edited settings and the same view.
- Seed the editor so the primary workflow is usable on first load with sensible defaults (font-size 13, background #282c34, foreground #ffffff, the standard 16-color palette, and the default keybind set), and so the generated-config panel starts from a known empty state.
- Provide the generated-configuration output as monospace key = value text with the two header comment lines and an override count, an empty state when nothing is overridden, and Copy, Download, and Reset all actions.
- Deleting or resetting an override must remove exactly that override; it must not clear or alter unrelated overrides.
- Do not fabricate success: Copy/Download/Reset and every setting edit must reflect real state, differ for different inputs, and survive a reload.
</requirements>
